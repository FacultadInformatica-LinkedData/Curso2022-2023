URI Strategy


Content Negotiation:

The application will handle content negotiation very simple. When requesting resources with a slash URI, a JSON file will be returned whilst a
turtle file with the requested ontology will be what a hash URI returns


# Pattern URI for resources:
http://www.semanticweb_g27.org/resource/{Class_name}/{resource_id}

# Pattern URI for Classes and Properties:
http://www.semanticweb_g27.org/ontology/AirQuality#[{Class_name} | {Property_name}]

############
# Ontology URIs #
############

##
### Properties Uris:
##
http://www.semanticweb_g27.org/ontology/AirQuality#isValid
http://www.semanticweb_g27.org/ontology/AirQuality#magnitude

##
### Class Uris:
##

http://www.semanticweb_g27.org/ontology/AirQuality#Municipality
http://www.semanticweb_g27.org/ontology/AirQuality#Province

############
# Resource URIs #
############

##
### Instances
##

# Instance of Municipality example:
## resource identifier -> Using 'M'+ID provided by the dataset
http://www.semanticweb_g27.org/resource/Municipality/M28

# Instance of Province example:
## resource identifier -> Using 'P'+ID provided by the dataset
http://www.semanticweb_g27.org/resource/Province/P079

# Instance of CivicStructure example:
## resource identifier -> Using 'S'+ID provided by the dataset
http://www.semanticweb_g27.org/resource/CivicStructure/S28079004

# Instance of Observation example:
## resource identifier -> Using 'O'+ID provided by the dataset
http://www.semanticweb_g27.org/resource/Observation/O28079004_1_38

# Instance of GeoCoordinates example:
## resource identifier -> Using 'GEO_'+Related_Object_ID
http://www.semanticweb_g27.org/resource/GeoCoordinates/GEO_M28
